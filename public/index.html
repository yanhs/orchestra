Agent Orchestra

Multi-agent orchestration — Claude agents with different roles discuss, develop, and review together.

Interaction Modes

Discussion

Round-robin debate. Each agent sees the full transcript and adds their perspective. PM summarizes.

Topic → A → B → C → A → … → Summary

Pipeline

Sequential handoff. Design → code → review → test. Rework loop on critical issues.

Architect → Developer → Reviewer → Tester

Parallel

Fan-out / fan-in. Agents work simultaneously on different aspects, then results are merged.

Task → [A, B, C] → Merge

Consensus

Voting with structured output. Agents vote independently, supermajority decides.

Question → All vote → Tally → Decision

Agent Roles

Architect opus
Developer sonnet
Reviewer opus
PM sonnet
Tester sonnet
DevOps sonnet
Security opus

Usage

# Discussion
orchestra discuss "REST or GraphQL?" --agents architect,developer,reviewer

# Pipeline
orchestra pipeline "Add authentication" --steps architect:design,developer:implement,reviewer:review

# Parallel
orchestra parallel "Build dashboard" --tasks "developer:frontend" "developer:API" "devops:Docker"

# Consensus
orchestra consensus "PostgreSQL or MongoDB?" --agents architect,developer,devops

Architecture

Each agent is an independent ClaudeSDKClient process. Python orchestrator controls turn order, message routing, and stopping criteria. No shared context window — each agent gets its full capacity.

Runs on Claude Max subscription (no per-token API costs). Roles, modes, and prompts are configurable via YAML.
