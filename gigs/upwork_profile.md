### What I Actually Do

**Multi-Agent Orchestration**
I design systems where multiple AI agents coordinate — one plans, others execute, results get aggregated. This isn't just "calling GPT twice." It's proper task decomposition, inter-agent communication, error recovery, and parallel execution with controlled concurrency. If you've got a complex workflow that a single prompt can't handle, this is my zone.

**Claude API & OpenAI Integration**
Deep experience with Anthropic's Claude API — tool use, streaming, structured outputs, system prompt engineering, multi-turn context management. Same depth with OpenAI GPT-4. I know the quirks of both models and how to get consistent, predictable behavior from them in production.

**FastAPI + WebSocket Backends**
Every AI agent needs an API layer. I build them with FastAPI — async, performant, properly typed. WebSocket support for real-time streaming so your users see AI responses as they generate, not waiting for a wall of text. Proper connection lifecycle management, not a quick hack.

**RAG Pipelines**
Connect your internal documents, databases, or knowledge bases to an LLM that can actually reason over them. Vector embeddings, semantic search, context injection, source citation. Built with LangChain or from scratch depending on your needs.

**Automation & Integration**
AI-powered automation that replaces manual processes: email processing agents, data enrichment pipelines, Slack/Telegram bots with real business logic, webhook processors, scheduled AI tasks. If a human is doing something repetitive that involves reading and making decisions — I can automate it.
